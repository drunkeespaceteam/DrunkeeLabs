====================================================
CRITICAL PREVIEW FIX (MOST IMPORTANT)
=====================================

CURRENT ISSUE:
Sometimes one employee sees another employee preview or fallback NexusDev UI because preview mappings are colliding.

REQUIRED FIX:
Implement immutable submission-scoped preview architecture.

REQUIREMENTS:

Each submission must generate:

* unique submissionId
* unique preview URL
* unique Docker container
* unique dynamic port
* unique Redis mapping

DO NOT USE:

* preview:userId
* active-preview
* global preview mappings

USE:
preview:{submissionId}

Preview URL format:
"/preview/:submissionId"

Container naming:
"nexus-sub-{submissionId}"

Redis structure:
preview:{submissionId} => {
submissionId,
userId,
containerId,
port,
runtime,
createdAt,
status
}

Add Redis TTL cleanup:
EX: 3600

====================================================
PROXY ROUTING FIX
=================

Implement submission-scoped reverse proxy routing.

Use:
http-proxy-middleware

Flow:

1. Read submissionId from route
2. Fetch Redis mapping
3. Route to correct container port
4. Never reuse preview mappings

Add:
Cache-Control: no-store

SUBMISSION SNAPSHOT STORAGE SYSTEM
==================================

CURRENT ISSUE:

REQUIRED FIX:
Store immutable submission snapshots in database/storage when employee submits.

NEW FLOW:

1. Employee uploads ZIP/project
2. Backend creates permanent submission snapshot
3. Store:

   * ZIP file
   * extracted source snapshot
   * metadata
   * runtime
   * logs
   * preview URL
4. Keep snapshot permanently
5. Never delete winner snapshots

===================
